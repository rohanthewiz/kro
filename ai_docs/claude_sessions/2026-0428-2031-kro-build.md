## Phases

### 1. Plan mode (initial scope)

User asked for a standalone app to monitor/manage k8s resources in `~/.kube/config`,
starting from the existing `coord/k8s_resources*.go` and `web/k8s_resources.go` files
in edp_dataflow. Mid-plan they added: "I would like to quickly switch
namespaces/contexts."

Two parallel Explore agents inventoried the source code and edp_dataflow's web
patterns (rweb routes, element HTML builder, SSE hub, config loading).

Four AskUserQuestion choices locked:
- Kubeconfig source: load `~/.kube/config` on startup (no paste UI)
- Namespace UX: dropdown of accessible namespaces with free-text fallback
- Operations scope: expanded read-only (existing 4 kinds + Services / StatefulSets /
  DaemonSets / Ingresses / ConfigMaps / Secrets)
- Selection persistence: browser cookies (`kro_ctx`, `kro_ns`)

### 2. Initial implementation

Layout:

```
config/   env-driven config
kube/     loader, registry, selector, resources, describe, logs, delete, pod_info
web/      server, handlers, sse, pages, header_component, embeds/
main.go
```

Key design decisions:

- **`kube.ClientRegistry`** — lazy per-context `*kubernetes.Clientset` cache built via
  `clientcmd.NewNonInteractiveClientConfig`. Replaces the SHA256-keyed cache in the
  edp_dataflow source.
- **`kube.ResolveSelection(c, reg, pinnedFn)`** — single resolver: cookie → first
  pinned → kubeconfig default → "default".
- **Per-client SSE goroutine** (`web/sse.go`) — each connection gets its own snapshot
  loop reading the cookie-resolved (ctx, ns). `rweb.SSEHub` was rejected because its
  fan-out broadcasts identical payloads, which doesn't fit per-tab cluster targeting.
- **Resource hierarchy** extended for StatefulSets / DaemonSets owner-ref assignment
  alongside the existing Job/Deployment/ReplicaSet logic.

`go mod tidy` pulled k8s.io/{api,apimachinery,client-go} v0.35.0. Built clean.

Tests: `kube/registry_test.go` (4 tests on `Contexts()`, `HasContext`,
`DefaultNamespace`, lazy client caching), `kube/selector_test.go` (3 tests via
rweb's in-memory `Request()` test client).

### 3. New requirement: pinned namespaces per cluster

User said: "I want to dynamically add namespaces per cluster somewhat similar to
adding a category to an article, but just one selectable at a time."

Two preview-style questions confirmed: replace the live `Namespaces.List`
(rather than coexist), and persist server-side as a local JSON file.

Built **`state/store.go`** — file-backed JSON at
`os.UserConfigDir()/kro/state.json` (overridable via `KRO_STATE_FILE`). Atomic
writes (temp + rename), dedup, sorted, version-tagged. 4 unit tests covering
add/remove/persist/missing-file.

Wired through:

- `kube/namespaces.go` deleted (live List dropped).
- `ResolveSelection` extended with a `PinnedFn` parameter.
- New endpoints: `POST /api/namespaces` (add, optionally `select: true`),
  `DELETE /api/namespaces?name=...` (with cookie fallback when active was removed).
- `GET /api/namespaces` auto-pins the kubeconfig default on first encounter so the
  dropdown is never empty for a known cluster.
- Header UI: `+` button reveals an inline input (Enter adds, Esc cancels);
  `×` confirms-and-removes the currently selected namespace.

Smoke test against the real kubeconfig confirmed: auto-pin → add → dedup → remove →
JSON file ends up with the expected sorted list.

### 4. Section collapse/expand + branding tweaks

Refactored the JS section builders (`sectionHierarchical`, `sectionDeployments`,
`flatSection`, plus the inline orphans/all-pods sections) to all funnel through a
shared `sectionShell(slug, title, count, body)` helper. Each section is now a
`<div class="resource-section" data-section="<slug>">` with a clickable
`.section-header` and a `▸` chevron. State persists per browser via
`localStorage["kro_collapsed_<slug>"]`. Initial collapsed state is set during
`rebuildTables` so the 10s SSE re-renders never flash open-then-collapse.

Branding rename: "kro" → "KRo" in the page title, header bar title, and JS
comment/dialog text.

## Notes / non-obvious calls

- **rweb cookie HttpOnly default**: `ctx.SetCookie(name, value)` defaults
  HttpOnly=true. We rely on the server reading cookies (not JS), so this is fine.
- **rweb's SSEHub doesn't support per-client topic filtering** in the pinned
  version, so we wired `Server.SetupSSE` directly with a per-connection goroutine
  rather than fighting the hub.
- **Auto-pinning the kubeconfig default** in `GET /api/namespaces` (rather than in
  the selector) keeps `ResolveSelection` pure and avoids writes on every request.
- **Section slugs** are hand-picked stable identifiers (`jobs`, `deployments`,
  `statefulsets`, `daemonsets`, `pods-orphan`, `all-pods`, `services`, `ingresses`,
  `configmaps`, `secrets`) — renaming them would invalidate users' localStorage
  collapsed-state.
