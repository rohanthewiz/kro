## How theming works (context)

- Theme toggle adds/removes class `dark` on `<body>` (`toggleDarkMode()` in
  `web/embeds/resources.js`), persisted in `localStorage['darkMode']`.
- Convention: **base rules = light**, `body.dark .x { ... }` = dark overrides.
- Exception: the kubectl **terminal** (`.term-*`, `.tk-*`) was authored *dark* in
  the base rules and shared by both themes. To re-skin it for light without
  touching dark mode, light overrides were scoped under `body:not(.dark)` (mirrors
  the real toggle, wins on specificity). Dark mode left byte-for-byte unchanged.

## Key lesson / method

First two attempts nudged individual hex values per-element → the page drifted
**green**, the terminal drifted **blue**, surfaces were near-white blanks → "not
jiving." Fix was to work from **one undertone**: cyan sits between green and blue,
so a single cyan/teal family satisfies both "green-cyan page" and "bluish-gray
terminal" while making everything relatives. After that, direction tweaks
(lighten / shift hue) are one coordinated move instead of more guessing.
